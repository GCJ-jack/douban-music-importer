# Douban Music New Subject Form Research

This document records observations from an authenticated Chrome session. It is research input for the v0.1 Douban Form Assistant, not a final selector contract.

## Confirmed URLs

- `https://music.douban.com/new_subject`
- `https://music.douban.com/new_subject?cat=1003&search_text=<query>`

The new-subject flow is not a single form in the common case. It starts with a short lookup / duplicate-check step, then can proceed to a detailed metadata form.

## Step 1: Lookup / Duplicate Check

Observed heading:

- `添加新的唱片`

Observed visible fields:

- `唱片名`
- `条形码`

Observed actions:

- `下一步`
- `取消`
- `添加无条形码的唱片`

Product implication:

- Many real-world album additions may not have a barcode available.
- v0.1 should treat `添加无条形码的唱片` as a common path, not an exception.
- Barcode should be imported when Discogs provides it, but lack of barcode should not block the Douban-assisted flow.
- The assistant should not click `下一步` or `添加无条形码的唱片` automatically. It may explain which step the user is on and wait for user action.

- `添加Tony! Toni! Toné!`

Observed visible fields:

- `唱片名`
- `又名`
- `表演者`

- `介质`

- `出版者 (必填)`
- `碟片数`
- `ISRC`
- `曲目`
- `简介`
- `参考资料 (必填)`

Observed action:

- `下一步`
- `取消`

Field behavior seen from the page:

- `又名` supports adding multiple values with a `+` control.
- `表演者` supports multiple rows and a `+` control.
- `流派`, `专辑类型`, and `介质` are selection controls.
- `曲目` is a multiline textarea; one track title per line.
- `参考资料` is required and should be a good place for source links / attribution, subject to final selector confirmation.

`又名`:

- First observed input: `#p_56_0`, `name="p_56"`.
- `+` control observed as `a.add`.
- v0.1 can fill the first safe input only. Dynamic row creation should be treated cautiously and can be deferred.

`表演者`:

- Observed inputs: `#p_48_0`, `#p_48_1`, `#p_48_2`, all `name="p_48"`.
- Each row also has a hidden `m_48` input.

- v0.1 can fill existing visible `input[name="p_48"]` rows only. It must not modify hidden `m_48` fields.

### System fields and submit controls

Observed system / workflow fields:

- `input[name="ck"]`
- `input[name="cat"]`, value `1003`
- `input[name="no_uid"]`, value `yes`
- hidden `input[name="search_text"]`, value matching the searched title
- hidden `input[name="m_48"]` rows for performer metadata

The form assistant must not modify these system fields.

Observed submit control:

- `input[name="detail_subject_submit"]`, value `下一步`

The form assistant must not click `detail_subject_submit` and must not submit the form.

### Detection notes

Detailed form detection can use:

- URL path: `/new_subject`
- Page title: `新添唱片`
- Heading beginning with `添加`
- Presence of detailed-form fields such as `#p_27`, `#p_51`, `#p_50`, `textarea[name="p_52_other"]`, and `textarea[name="p_152_other"]`
- Presence of `input[name="detail_subject_submit"]`

Lookup / duplicate-check page detection should remain separate and should continue to rely on the lookup-specific fields and actions observed earlier:

- Heading such as `添加新的唱片`
- `唱片名`
- `条形码`
- `添加无条形码的唱片`

## v0.1 Mapping Notes

Likely high-value draft mappings:

- Discogs release title -> `唱片名`
- Alternative / translated titles -> `又名`
- Main artists -> `表演者`
- Genres / styles -> `流派` with review required
- Formats -> `专辑类型` / `介质` with review required
- Release date -> `发行时间`
- Labels -> `出版者`
- Format quantity -> `碟片数`
- ISRC identifiers -> `ISRC`, if available
- Tracklist -> `曲目`
- Discogs release URL and attribution -> `参考资料`
- Notes -> `简介`, with review required

## Remaining Gaps

Remaining gaps before or during #8 implementation:

- Exact option values for `流派`, `专辑类型`, and `介质`.
- Whether the detailed form URL changes after choosing the no-barcode path.
- Validation behavior for required fields.
- Whether `参考资料` accepts plain source URLs cleanly.
- Whether barcode path and no-barcode path produce the same detailed form.

#8 first pass should stay conservative: support only safe text inputs and textareas, keep custom selects as review-only / unsupported, never overwrite existing values by default, never modify system fields, and never submit the form.
